# 테이블오더 서비스 - 요구사항 정의서

## Intent Analysis

- **User Request**: 테이블오더 서비스 Backend API 구축
- **Request Type**: New Project (Greenfield)
- **Scope Estimate**: System-wide (멀티 매장 지원, 고객/관리자 기능 전체)
- **Complexity Estimate**: Complex (다수의 도메인, 실시간 통신, 세션 관리)
- **Requirements Depth**: Comprehensive

## Functional Requirements

### FR-1: 매장 관리 (Store Management)

- **FR-1.1**: 멀티 매장 지원 - 여러 매장을 독립적으로 관리
- **FR-1.2**: 매장별 고유 식별자 (storeId) 기반 데이터 격리
- **FR-1.3**: 관리자 계정은 사전 등록 방식 (DB seed)

### FR-2: 인증 및 세션 관리 (Authentication & Session)

#### FR-2.1: 관리자 인증
- 매장 식별자 + 사용자명 + 비밀번호 기반 로그인
- JWT 토큰 발급 (16시간 유효)
- 비밀번호 bcrypt 해싱
- 로그인 시도 제한

#### FR-2.2: 테이블 태블릿 인증
- 매장 식별자 + 테이블 번호 + 테이블 비밀번호 기반 로그인
- 로그인 성공 시 JWT 토큰 발급
- 자동 로그인을 위한 토큰 관리 (클라이언트 측)

### FR-3: 메뉴 관리 (Menu Management)

- **FR-3.1**: 메뉴 CRUD (등록, 조회, 수정, 삭제)
- **FR-3.2**: 카테고리별 메뉴 분류
- **FR-3.3**: 메뉴 정보: 메뉴명, 가격, 설명, 카테고리, 이미지
- **FR-3.4**: 메뉴 노출 순서 조정
- **FR-3.5**: 이미지 파일 업로드 (서버 로컬 파일 시스템 저장)
- **FR-3.6**: 필수 필드 및 가격 범위 검증

### FR-4: 주문 처리 (Order Processing)

- **FR-4.1**: 주문 생성 - 매장ID, 테이블ID, 메뉴 목록(메뉴명, 수량, 단가), 총 금액, 세션ID 포함
- **FR-4.2**: 주문 상태 관리 - 대기중(PENDING) / 준비중(PREPARING) / 완료(COMPLETED)
- **FR-4.3**: 주문 상태 변경 API (관리자용)
- **FR-4.4**: 주문 삭제 API (관리자 직권 수정)
- **FR-4.5**: 테이블별 현재 세션 주문 내역 조회
- **FR-4.6**: 주문 시간 순 정렬

### FR-5: 테이블 세션 관리 (Table Session)

- **FR-5.1**: 테이블 세션 시작 - 해당 세션의 첫 주문 시 자동 생성
- **FR-5.2**: 테이블 세션 종료 (이용 완료) - 관리자가 수동 처리
- **FR-5.3**: 세션 종료 시 주문 내역을 과거 이력(OrderHistory)으로 이동
- **FR-5.4**: 세션 종료 시 테이블 현재 주문 목록 및 총 주문액 리셋
- **FR-5.5**: 새 고객은 이전 주문 내역 없이 시작

### FR-6: 실시간 주문 모니터링 (Real-time Monitoring)

- **FR-6.1**: SSE (Server-Sent Events) 기반 실시간 주문 알림
- **FR-6.2**: 신규 주문 발생 시 2초 이내 관리자 화면 전달
- **FR-6.3**: 주문 상태 변경 시 실시간 업데이트
- **FR-6.4**: 매장별 SSE 스트림 (storeId 기반 필터링)

### FR-7: 과거 주문 내역 (Order History)

- **FR-7.1**: 테이블별 과거 주문 목록 조회 (시간 역순)
- **FR-7.2**: 주문 정보: 주문 번호, 시각, 메뉴 목록, 총 금액, 이용 완료 시각
- **FR-7.3**: 날짜 필터링 기능

## Non-Functional Requirements

### NFR-1: 성능
- SSE 이벤트 전달: 2초 이내
- API 응답 시간: 일반 API 500ms 이내

### NFR-2: 보안
- 비밀번호 bcrypt 해싱
- JWT 토큰 기반 인증 (16시간 만료)
- 로그인 시도 제한
- 매장별 데이터 격리 (storeId 기반)

### NFR-3: 데이터 무결성
- 주문 생성 시 트랜잭션 처리
- 세션 종료 시 주문 이력 이동 트랜잭션 처리
- 필수 필드 검증

### NFR-4: API 문서화
- Swagger (OpenAPI 3.0) 기반 API 문서 자동 생성
- 모든 API endpoint 문서화

## 제외 범위 (Out of Scope)

- 결제 처리 (PG사 연동, 영수증, 환불, 포인트/쿠폰)
- 복잡한 인증 (OAuth, SNS 로그인, 2FA)
- 이미지 리사이징/최적화
- 알림 시스템 (푸시, SMS, 이메일)
- 주방 기능 (주방 전달, 재고 관리)
- 고급 기능 (분석, 매출 리포트, 직원 관리, 예약, 리뷰, 다국어)
- 외부 연동 (배달 플랫폼, POS, 소셜 미디어)
- Frontend 구현 (별도 프로젝트)

## MVP 범위 요약

### 고객용 API
1. 테이블 태블릿 로그인
2. 메뉴 조회 (카테고리별)
3. 주문 생성
4. 현재 세션 주문 내역 조회

### 관리자용 API
1. 관리자 로그인
2. 메뉴 CRUD + 이미지 업로드
3. 실시간 주문 모니터링 (SSE)
4. 주문 상태 변경
5. 주문 삭제
6. 테이블 세션 관리 (이용 완료 처리)
7. 과거 주문 내역 조회
