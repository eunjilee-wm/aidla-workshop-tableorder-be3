# User Personas

## Persona 1: 고객 (Customer)

| 항목 | 내용 |
|------|------|
| 이름 | 김고객 |
| 역할 | 매장 방문 고객 |
| 접근 방식 | 모바일 최적화 웹 (Mobile Web) |
| 목표 | 빠르고 편리하게 음식을 주문하고, 주문 상태를 확인 |
| 특성 | QR코드 스캔으로 즉시 접근, 로그인 불필요, 앱 설치 불필요, 모바일 터치 최적화 UI |
| Pain Points | 직원 호출 대기 시간, 주문 누락, 주문 상태 불확실, 작은 화면에서의 조작 |

## Persona 2: 매장 관리자 (Store Admin)

| 항목 | 내용 |
|------|------|
| 이름 | 이관리 |
| 역할 | 매장 운영 관리자 |
| 접근 방식 | PC/태블릿 웹 브라우저 |
| 목표 | 실시간 주문 관리, 메뉴 운영, 테이블 세션 관리를 통한 효율적 매장 운영 |
| 특성 | 16시간 장시간 세션, 다수 테이블 동시 관리 |
| Pain Points | 주문 누락, 테이블 상태 파악 어려움, 수동 메뉴 관리 번거로움 |
