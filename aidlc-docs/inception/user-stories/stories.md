# User Stories

분류 방식: Feature-Based | Acceptance Criteria: 상세 (Given-When-Then) | 우선순위: Must/Should/Could

---

## 1. 인증 (Authentication)

### US-1.1: QR코드 기반 메뉴 접근
- **Persona**: 고객 (김고객)
- **Priority**: Must
- **Story**: 고객으로서, 테이블의 QR코드를 스캔하여 로그인 없이 즉시 메뉴 화면에 접근할 수 있다.

**Acceptance Criteria**:
- Given 고객이 테이블에 앉았을 때, When QR코드를 스캔하면, Then 매장ID와 테이블 번호가 포함된 모바일 웹 메뉴 화면으로 바로 이동한다.
- Given QR코드 URL에 유효한 매장ID와 테이블 번호가 포함되어 있을 때, When 페이지가 로드되면, Then 해당 매장의 메뉴가 표시된다.
- Given QR코드 URL에 유효하지 않은 매장ID 또는 테이블 번호가 포함되어 있을 때, When 페이지가 로드되면, Then 에러 메시지가 표시된다.
- Given 고객이 메뉴 화면에 접근했을 때, When 별도 로그인 절차 없이, Then 메뉴 조회와 주문이 가능하다.
- Given 고객이 모바일 브라우저에서 접근했을 때, When 화면이 렌더링되면, Then 모바일에 최적화된 UI가 표시된다.

### US-1.2: 관리자 로그인
- **Persona**: 매장 관리자 (이관리)
- **Priority**: Must
- **Story**: 관리자로서, 매장 식별자와 계정 정보로 로그인하여 매장 관리 시스템에 접근할 수 있다.

**Acceptance Criteria**:
- Given 관리자가 로그인 화면에 접근했을 때, When 매장 식별자, 사용자명, 비밀번호를 입력하고 로그인하면, Then 16시간 유효한 JWT 토큰이 발급되고 대시보드로 이동한다.
- Given 유효하지 않은 인증 정보를 입력했을 때, When 로그인을 시도하면, Then 에러 메시지가 표시된다.
- Given 로그인 세션이 유효할 때, When 브라우저를 새로고침하면, Then 세션이 유지된다.
- Given 16시간이 경과했을 때, When API를 호출하면, Then 자동 로그아웃되고 로그인 화면으로 이동한다.
- Given 로그인 시도가 연속으로 실패했을 때, When 제한 횟수를 초과하면, Then 일정 시간 동안 로그인이 차단된다.

## 2. 메뉴 조회 (Menu Browsing)

### US-2.1: 카테고리별 메뉴 조회
- **Persona**: 고객 (김고객)
- **Priority**: Must
- **Story**: 고객으로서, 모바일 화면에서 카테고리별로 메뉴를 탐색하여 원하는 음식을 쉽게 찾을 수 있다.

**Acceptance Criteria**:
- Given 고객이 QR코드로 접근한 상태일 때, When 메뉴 화면이 로드되면, Then 카테고리 목록과 해당 카테고리의 메뉴가 모바일에 최적화된 레이아웃으로 표시된다.
- Given 메뉴 목록이 표시된 상태일 때, When 특정 카테고리를 선택하면, Then 해당 카테고리의 메뉴만 필터링되어 표시된다.
- Given 메뉴가 표시된 상태일 때, When 메뉴 항목을 확인하면, Then 메뉴명, 가격, 설명, 이미지가 표시된다.
- Given 메뉴가 카테고리별로 표시될 때, When 카테고리 간 이동하면, Then 빠르게 전환된다.
- Given 매장에 등록된 메뉴가 없을 때, When 메뉴 화면에 접근하면, Then 적절한 안내 메시지가 표시된다.

## 3. 장바구니 (Cart)

### US-3.1: 장바구니 관리
- **Persona**: 고객 (김고객)
- **Priority**: Must
- **Story**: 고객으로서, 모바일 웹에서 선택한 메뉴를 장바구니에 담고 수정할 수 있다.

**Acceptance Criteria**:
- Given 메뉴 화면에서 메뉴를 선택했을 때, When 장바구니에 추가하면, Then 해당 메뉴가 장바구니에 추가되고 총 금액이 업데이트된다.
- Given 장바구니에 메뉴가 있을 때, When 수량을 증가/감소하면, Then 수량과 총 금액이 실시간으로 재계산된다.
- Given 장바구니에 메뉴가 있을 때, When 특정 메뉴를 삭제하면, Then 해당 메뉴가 제거되고 총 금액이 재계산된다.
- Given 장바구니에 메뉴가 있을 때, When 장바구니 비우기를 실행하면, Then 모든 메뉴가 제거되고 총 금액이 0이 된다.
- Given 장바구니에 메뉴를 담은 상태에서, When 모바일 브라우저를 새로고침하면, Then 장바구니 내용이 유지된다 (로컬 저장).
- Given 수량이 1인 메뉴에서, When 수량을 감소하면, Then 해당 메뉴가 장바구니에서 제거된다.

## 4. 주문 (Order)

### US-4.1: 주문 생성
- **Persona**: 고객 (김고객)
- **Priority**: Must
- **Story**: 고객으로서, 장바구니의 메뉴를 주문하여 매장에 전달할 수 있다.

**Acceptance Criteria**:
- Given 장바구니에 메뉴가 있을 때, When 주문 확정 버튼을 누르면, Then 주문이 대기중(PENDING) 상태로 생성되고 주문 번호가 표시된다.
- Given 주문이 성공적으로 생성되었을 때, When 주문 완료 화면이 표시되면, Then 5초 후 메뉴 화면으로 자동 리다이렉트된다.
- Given 주문이 성공했을 때, When 주문이 완료되면, Then 장바구니가 자동으로 비워진다.
- Given 주문 생성 중 서버 오류가 발생했을 때, When 주문이 실패하면, Then 에러 메시지가 표시되고 장바구니가 유지된다.
- Given 장바구니가 비어있을 때, When 주문 확정을 시도하면, Then 주문이 차단되고 안내 메시지가 표시된다.
- Given 현재 테이블에 활성 세션이 없을 때, When 첫 주문이 생성되면, Then 서버에서 매장ID+테이블번호 기반으로 새로운 테이블 세션이 자동으로 시작된다.

### US-4.2: 주문 내역 조회
- **Persona**: 고객 (김고객)
- **Priority**: Must
- **Story**: 고객으로서, 모바일 웹에서 현재 테이블의 주문 내역과 승인/거절 상태를 확인할 수 있다.

**Acceptance Criteria**:
- Given 고객이 주문 내역 화면에 접근했을 때, When 주문 내역을 조회하면, Then 현재 세션의 주문만 시간 순으로 표시된다.
- Given 주문 내역이 표시될 때, When 각 주문을 확인하면, Then 주문 번호, 시각, 메뉴/수량, 금액, 상태(대기중/승인/거절/준비중/완료)가 표시된다.
- Given 이전 세션(이용 완료 처리된)의 주문이 있을 때, When 주문 내역을 조회하면, Then 이전 세션 주문은 표시되지 않는다.
- Given 현재 세션에 주문이 없을 때, When 주문 내역을 조회하면, Then 적절한 안내 메시지가 표시된다.
- Given 주문이 관리자에 의해 거절되었을 때, When 주문 내역을 조회하면, Then 해당 주문이 거절 상태와 거절 사유(있는 경우)로 표시된다.

## 5. 주문 모니터링 (Order Monitoring)

### US-5.1: 주문 모니터링
- **Persona**: 매장 관리자 (이관리)
- **Priority**: Must
- **Story**: 관리자로서, 들어오는 주문을 확인하여 승인 또는 거절할 수 있다.

**Acceptance Criteria**:
- Given 관리자가 대시보드에 접속한 상태일 때, When 주문 목록을 조회하면, Then 매장의 현재 주문이 표시된다.
- Given 주문 목록이 표시될 때, When 각 주문을 확인하면, Then 테이블 번호, 주문 번호, 시각, 메뉴/수량(축약), 총 금액이 표시된다.
- Given 테이블별 카드가 표시될 때, When 카드를 확인하면, Then 해당 테이블의 총 주문액과 최신 주문 미리보기가 표시된다.
- Given 주문 카드가 표시될 때, When 카드를 클릭하면, Then 전체 메뉴 목록 상세 보기가 표시된다.
- Given 주문 상태별 필터가 있을 때, When 특정 상태를 선택하면, Then 해당 상태의 주문만 표시된다.

### US-5.2: 주문 승인/거절
- **Persona**: 매장 관리자 (이관리)
- **Priority**: Must
- **Story**: 관리자로서, 신규 주문을 확인하고 승인 또는 거절하여 주문 진행 여부를 결정할 수 있다.

**Acceptance Criteria**:
- Given 대기중(PENDING) 상태의 주문이 있을 때, When 승인 버튼을 누르면, Then 주문 상태가 ACCEPTED로 변경된다.
- Given 대기중(PENDING) 상태의 주문이 있을 때, When 거절 버튼을 누르면, Then 주문 상태가 REJECTED로 변경된다.
- Given 주문을 거절할 때, When 거절 사유를 입력하면, Then 거절 사유가 주문에 저장된다.
- Given 주문이 승인/거절되었을 때, When 관리자가 주문 목록을 새로고침하면, Then 변경된 상태가 반영된다.
- Given 이미 승인/거절된 주문일 때, When 다시 승인/거절을 시도하면, Then 변경이 차단되고 안내 메시지가 표시된다.
- Given 주문 승인/거절 중 오류가 발생했을 때, When 처리가 실패하면, Then 에러 메시지가 표시되고 이전 상태가 유지된다.

### US-5.3: 주문 상태 변경
- **Persona**: 매장 관리자 (이관리)
- **Priority**: Must
- **Story**: 관리자로서, 승인된 주문의 상태를 변경하여 주문 처리 진행 상황을 관리할 수 있다.

**Acceptance Criteria**:
- Given 주문이 승인(ACCEPTED) 상태일 때, When 준비중으로 변경하면, Then 상태가 PREPARING으로 업데이트된다.
- Given 주문이 준비중(PREPARING) 상태일 때, When 완료로 변경하면, Then 상태가 COMPLETED로 업데이트된다.
- Given 주문 상태가 변경되었을 때, When 관리자가 주문 목록을 새로고침하면, Then 변경된 상태가 반영된다.
- Given 주문 상태 변경 중 오류가 발생했을 때, When 변경이 실패하면, Then 에러 메시지가 표시되고 이전 상태가 유지된다.
- Given 거절된 주문일 때, When 상태 변경을 시도하면, Then 변경이 차단되고 안내 메시지가 표시된다.

## 6. 테이블 관리 (Table Management)

### US-6.1: 주문 삭제
- **Persona**: 매장 관리자 (이관리)
- **Priority**: Must
- **Story**: 관리자로서, 잘못된 주문을 삭제하여 정확한 주문 관리를 할 수 있다.

**Acceptance Criteria**:
- Given 테이블에 주문이 있을 때, When 특정 주문의 삭제 버튼을 누르면, Then 확인 팝업이 표시된다.
- Given 확인 팝업이 표시된 상태에서, When 확인을 누르면, Then 주문이 즉시 삭제되고 테이블 총 주문액이 재계산된다 (승인/준비중/완료 상태 주문만 합산).
- Given 확인 팝업이 표시된 상태에서, When 취소를 누르면, Then 삭제가 취소되고 이전 상태가 유지된다.
- Given 주문 삭제가 성공했을 때, When 결과가 반환되면, Then 성공 피드백이 표시된다.
- Given 주문 삭제 중 오류가 발생했을 때, When 삭제가 실패하면, Then 실패 피드백이 표시되고 주문이 유지된다.

### US-6.2: 테이블 이용 완료 (세션 종료)
- **Persona**: 매장 관리자 (이관리)
- **Priority**: Must
- **Story**: 관리자로서, 고객의 식사가 끝나면 테이블 이용 완료 처리를 하여 새 고객을 받을 수 있다.

**Acceptance Criteria**:
- Given 테이블에 활성 세션이 있을 때, When 이용 완료 버튼을 누르면, Then 확인 팝업이 표시된다.
- Given 확인 팝업에서 확인을 누르면, When 세션 종료가 처리되면, Then 해당 세션의 주문 내역이 과거 이력(OrderHistory)으로 이동된다.
- Given 세션 종료가 완료되면, When 테이블 상태를 확인하면, Then 현재 주문 목록과 총 주문액이 0으로 리셋된다.
- Given 세션이 종료된 테이블에서, When 새 고객이 첫 주문을 하면, Then 이전 주문 내역 없이 새로운 세션이 시작된다.
- Given 세션 종료 중 오류가 발생했을 때, When 처리가 실패하면, Then 실패 피드백이 표시되고 세션이 유지된다.

### US-6.3: 과거 주문 내역 조회
- **Persona**: 매장 관리자 (이관리)
- **Priority**: Should
- **Story**: 관리자로서, 테이블의 과거 주문 내역을 조회하여 매장 운영 이력을 확인할 수 있다.

**Acceptance Criteria**:
- Given 관리자가 테이블 관리 화면에 있을 때, When 과거 내역 버튼을 누르면, Then 해당 테이블의 과거 주문 목록이 시간 역순으로 표시된다.
- Given 과거 주문 목록이 표시될 때, When 각 주문을 확인하면, Then 주문 번호, 시각, 메뉴 목록, 총 금액, 이용 완료 시각이 표시된다.
- Given 과거 주문 목록이 표시될 때, When 날짜 필터를 적용하면, Then 해당 날짜 범위의 주문만 표시된다.
- Given 과거 내역 화면에서, When 닫기 버튼을 누르면, Then 대시보드로 복귀한다.
- Given 해당 테이블에 과거 주문이 없을 때, When 과거 내역을 조회하면, Then 적절한 안내 메시지가 표시된다.

## 7. 메뉴 관리 (Menu Management)

### US-7.1: 메뉴 CRUD
- **Persona**: 매장 관리자 (이관리)
- **Priority**: Must
- **Story**: 관리자로서, 메뉴를 등록/수정/삭제하여 매장 메뉴를 관리할 수 있다.

**Acceptance Criteria**:
- Given 관리자가 메뉴 관리 화면에 접근했을 때, When 메뉴 목록을 조회하면, Then 카테고리별로 메뉴가 표시된다.
- Given 메뉴 등록 화면에서, When 메뉴명, 가격, 설명, 카테고리, 이미지를 입력하고 저장하면, Then 새 메뉴가 등록된다.
- Given 필수 필드(메뉴명, 가격, 카테고리)가 비어있을 때, When 저장을 시도하면, Then 검증 에러 메시지가 표시된다.
- Given 가격이 유효 범위를 벗어났을 때, When 저장을 시도하면, Then 가격 범위 검증 에러가 표시된다.
- Given 기존 메뉴를 선택했을 때, When 정보를 수정하고 저장하면, Then 메뉴 정보가 업데이트된다.
- Given 기존 메뉴를 선택했을 때, When 삭제를 실행하면, Then 확인 후 메뉴가 삭제된다.

### US-7.2: 메뉴 이미지 업로드
- **Persona**: 매장 관리자 (이관리)
- **Priority**: Should
- **Story**: 관리자로서, 메뉴에 이미지를 업로드하여 고객에게 시각적 정보를 제공할 수 있다.

**Acceptance Criteria**:
- Given 메뉴 등록/수정 화면에서, When 이미지 파일을 선택하고 업로드하면, Then 이미지가 서버 로컬 파일 시스템에 저장되고 메뉴에 연결된다.
- Given 이미지가 업로드된 메뉴일 때, When 고객이 모바일 웹에서 메뉴를 조회하면, Then 이미지가 표시된다.
- Given 지원하지 않는 파일 형식을 업로드했을 때, When 업로드를 시도하면, Then 에러 메시지가 표시된다.
- Given 기존 이미지가 있는 메뉴에서, When 새 이미지를 업로드하면, Then 기존 이미지가 교체된다.
- Given 이미지 없이 메뉴를 등록할 때, When 저장하면, Then 기본 플레이스홀더 이미지로 표시된다.

### US-7.3: 메뉴 노출 순서 조정
- **Persona**: 매장 관리자 (이관리)
- **Priority**: Could
- **Story**: 관리자로서, 메뉴의 노출 순서를 조정하여 고객에게 원하는 순서로 메뉴를 보여줄 수 있다.

**Acceptance Criteria**:
- Given 메뉴 관리 화면에서, When 메뉴의 순서를 변경하면, Then 변경된 순서가 저장된다.
- Given 순서가 변경된 후, When 고객이 모바일 웹에서 메뉴를 조회하면, Then 변경된 순서로 메뉴가 표시된다.
- Given 같은 카테고리 내에서, When 순서를 변경하면, Then 해당 카테고리 내 순서만 변경된다.
- Given 새 메뉴가 등록되었을 때, When 순서가 지정되지 않으면, Then 해당 카테고리의 마지막에 추가된다.
- Given 메뉴가 삭제되었을 때, When 순서를 확인하면, Then 나머지 메뉴의 순서가 자동으로 재정렬된다.
